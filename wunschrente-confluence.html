1 Konzept und Zielsetzung

Der BVK Wunschrente-Rechner ist ein interaktives Vorsorge-Tool, das Versicherten ermöglicht, rückwärts zu rechnen: Ausgehend von einer gewünschten monatlichen Rente nach der Pensionierung wird berechnet, wie viel Gesamtkapital aus allen drei Säulen des Schweizer Vorsorgesystems benötigt wird — und ob dieses Kapital zum geplanten Rentenalter vorhanden ist.

1.1 Problemstellung

Das zentrale Problem besteht darin, dass Gelder aus den drei Säulen in unterschiedlichen Formen vorliegen:

SäuleForm der AngabeVerfügbarkeit
Säule 1 (AHV)Monatliche Rente in CHFVersicherter gibt Wert ein
Säule 2 (BVK)Gesamtkapital in CHFBVK liefert Wert via API
Säule 3 (privat)Gesamtkapital in CHFVersicherter gibt Wert ein

1.2 Lösungsansatz: Gemeinsamer Nenner

Alle drei Säulen werden auf eine monatliche Rente in CHF umgerechnet und summiert. Der Vergleich erfolgt direkt mit der eingegebenen Wunschrente.

Gesamtrente (mtl.) = AHV-Rente + BVK-Rente + Säule-3-Rente  →  Vergleich mit Wunschrente  →  Differenz = Lücke oder Überschuss

3 Berechnungslogik

3.1 Säule 1 – AHV-Rente

Die AHV-Rente wird vom Versicherten manuell eingegeben. Vier Optionen stehen zur Verfügung:

OptionWertStatus
Minimalrente (Standard)CHF 1’260 / MonatFrontend-Fixwert
Maximalrente (Einzelperson)CHF 2’520 / MonatFrontend-Fixwert
EhegattenrenteCHF 3’780 / MonatFrontend-Fixwert
Manuelle EingabeFreier CHF-BetragFrontend

Die AHV-Rente fliesst direkt als monatlicher CHF-Betrag in die Gesamtberechnung ein — keine Umrechnung notwendig.

💡 Hinweis: AHV-Rentenwerte

Die Fixwerte (CHF 1’260 / 2’520 / 3’780) basieren auf den aktuellen AHV-Gesetzesgrundlagen und sind jährlich auf Gültigkeit zu prüfen. Die Werte sind im Frontend hardcoded und müssen bei Anpassungen manuell aktualisiert werden.

Optional: Anbindung an e-AHVG API oder Empfehlung an Versicherten, den Wert von my.ahv.ch zu entnehmen.

3.2 Säule 2 – BVK-Rente

Das BVK-Sparguthaben (Gesamtkapital) wird via Umwandlungssatz in eine monatliche Rente umgerechnet:

BVK-Rente (mtl.) = Sparguthaben × Umwandlungssatz ÷ 12

3.2.1 Umwandlungssatz

Der Umwandlungssatz ist altersabhängig. Basis ist der intern gemessene BVK-Wert von 4.8% bei Rentenalter 65 (abgeleitet aus: CHF 36’000 Jahresrente bei CHF 750’000 Kapital). Die übrigen Altersstufen werden proportional skaliert:

RentenalterUmwandlungssatz (BVK intern, gerundet)Beispielrente bei CHF 750’000
60 Jahre~3.88%~CHF 2’425/Monat
62 Jahre~4.16%~CHF 2’600/Monat
65 Jahre (Basis)4.80%CHF 3’000/Monat
67 Jahre~5.22%~CHF 3’263/Monat
70 Jahre~5.79%~CHF 3’619/Monat
🔴 DUMMY → API ersetzen: Umwandlungssätze

Prototyp: Proportionale Skalierung ab BVG-Referenz (6.8% @ 65) auf BVK-intern (4.8% @ 65) gemäss fixer Tabelle im Frontend-Code.

Final: Gültige Umwandlungssätze direkt aus BVK-Reglement oder API beziehen.

GET /api/v1/conversion-rates

3.2.2 Beitragsvariante

Die Beitragsvariante (Basic / Standard / Top) beeinflusst das Sparguthaben durch höhere oder tiefere monatliche Beitragssätze (±5 Prozentpunkte je Stufe).

🔴 DUMMY → API ersetzen: Beitragsvariante-Einfluss

Prototyp: Basic = ×0.92 des Basisguthabens  |  Standard = ×1.0  |  Top = ×1.10 (pauschale Multiplikatoren).

Final: BVK-API liefert das hochgerechnete Guthaben für die gewählte Variante direkt als Kapitalwert. Der Multiplikator im Frontend fällt weg.

GET /api/v1/insured/{id}/projected-savings?retirementAge=65&contributionVariant=TOP

3.2.3 Freiwilliger Einkauf

Ein einmaliger Einkauf in die PK (heute getätigt) wächst bis zum Rentenalter durch Verzinsung an:

Einkauf-Zuwachs (Endwert) = Einkaufsbetrag × (1 + Zinssatz)35

🔴 DUMMY → API ersetzen: Einkauf-Berechnung

POST /api/v1/insured/{id}/voluntary-purchase-simulation

3.3 Säule 3 – Private Vorsorge

Das Kapital der Säule 3 liegt als Gesamtbetrag vor. Die Umrechnung erfolgt via vereinfachten Kapitalverzehr:

Säule-3-Rente (mtl.) = Kapital ÷ (Restlebensjahre × 12)
ParameterWertStatus
Lebenserwartung85 Jahre (fixiert)Vereinfacht
Restlebensjahre85 − RentenalterFrontend
StandardwertCHF 0 (leer)Frontend

Hinweis: Diese Methode ist konservativ und vereinfacht (kein Zinses-Zins auf verbleibendes Kapital). Für eine präzisere Berechnung wäre ein Annuitätenfaktor zu verwenden.

💡 Empfehlung: Kombinierter API-Endpunkt für alle Säule-2-Parameter

Ein einziger kombinierter Endpunkt für alle Säule-2-Parameter reduziert die Komplexität im Frontend erheblich:

POST /api/v1/insured/{id}/savings-simulation
{
  "retirementAge": 65,
  "contributionVariant": "TOP",
  "incomeScenario": "MODERATE",
  "voluntaryPurchase": 50000,
  "interestRate": 0.015
}
→ Response: { "projectedCapital": 850000, "monthlyPension": 3400 }

Die drei Empfehlungskacheln am Ende des Rechners generieren sich dynamisch basierend auf dem aktuellen Zustand:

💡 Produktive Logik (bleibt im Frontend)
  • AHV-Optionen: Fixwerte aus AHV-Gesetz (CHF 1’260 / 2’520 / 3’780)
  • Säule-3-Formel: Kapitalverzehr = Kapital ÷ (Restjahre × 12)
  • Lückenberechnung: Wunschrente − (AHV + BVK + S3) — reines Frontend
  • Empfehlungslogik: Dynamische Texte basierend auf aktuellem Zustand — reines Frontend

7 Technische Umsetzung

7.1 Frontend-Architektur

  • Vollständig eigenständiges HTML-Dokument (Single File, kein Framework)
  • Vanilla JS + HTML + CSS, alle Berechnungen im Browser
  • Fonts via Google Fonts CDN (Source Serif 4, DM Sans)
  • Responsive mit max-width 900px
  • Download-fähig als .html-Datei, direkt im Browser lauffähig

7.2 Confluence-Einbindung via GitHub Pages
